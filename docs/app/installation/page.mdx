# Installation

This guide will walk you through installing Notiq in your Next.js project.

## Prerequisites

Before installing Notiq, ensure your project meets the following requirements:

- **Node.js** 18.17 or later
- **React** 19.0.0 or later
- **React DOM** 19.0.0 or later
- **Next.js** 15.0.0 or later (App Router)

## Package Installation

### Using pnpm (Recommended)

```bash
pnpm add @collabchron/notiq
```

### Using npm

```bash
npm install @collabchron/notiq
```

### Using yarn

```bash
yarn add @collabchron/notiq
```

## Peer Dependencies

Notiq requires the following peer dependencies. If you're using Next.js 15+, these should already be installed:

```json
{
  "react": "^19.0.0",
  "react-dom": "^19.0.0",
  "next": "^15.0.0"
}
```

### Installing Peer Dependencies

If you need to install peer dependencies manually:

```bash
pnpm add react@19 react-dom@19 next@15
```

## Styling Setup

Notiq uses Tailwind CSS for styling. You need to import the Notiq styles in your application.

### 1. Import Notiq Styles

In your root layout file (`app/layout.tsx` or `app/layout.jsx`), import the Notiq CSS:

```tsx
import '@collabchron/notiq/dist/styles/notiq.css'
```

### 2. Configure Tailwind (Optional)

If you want to customize Notiq's styling, add the Notiq package to your Tailwind content paths:

```js
// tailwind.config.js
module.exports = {
  content: [
    './app/**/*.{js,ts,jsx,tsx,mdx}',
    './node_modules/@collabchron/notiq/dist/**/*.{js,jsx}',
  ],
  // ... rest of your config
}
```

## Font Setup (Optional)

Notiq uses KaTeX for mathematical equations, which requires specific fonts. The fonts are automatically included in the CSS bundle, but you may need to ensure your build process copies them correctly.

### Verifying Font Installation

After installation, verify that the KaTeX fonts are accessible:

1. Run your development server
2. Open the browser console
3. Check for any 404 errors related to font files

If you see font-related errors, ensure your Next.js configuration allows serving static assets from `node_modules`.

## TypeScript Support

Notiq is written in TypeScript and includes type definitions out of the box. No additional `@types` packages are required.

### Type Definitions

```tsx
import type { NotiqEditorProps } from '@collabchron/notiq'

const config: NotiqEditorProps = {
  // Your configuration with full type safety
}
```

## Verification

To verify that Notiq is installed correctly, create a simple test page:

```tsx
// app/test-editor/page.tsx
import { NotiqEditor } from '@collabchron/notiq'
import '@collabchron/notiq/dist/styles/notiq.css'

export default function TestPage() {
  return (
    <div className="p-4">
      <h1 className="text-2xl font-bold mb-4">Notiq Editor Test</h1>
      <NotiqEditor />
    </div>
  )
}
```

Navigate to `/test-editor` in your browser. If you see the editor interface, the installation was successful!

## Troubleshooting

### Module Not Found Error

If you encounter a "Module not found" error:

1. Clear your package manager cache:
   ```bash
   pnpm store prune
   # or
   npm cache clean --force
   ```

2. Delete `node_modules` and reinstall:
   ```bash
   rm -rf node_modules
   pnpm install
   ```

### CSS Not Loading

If styles aren't applying:

1. Ensure you've imported the CSS file in your root layout
2. Check that your build tool supports CSS imports
3. Verify there are no CSS module conflicts

### TypeScript Errors

If you see TypeScript errors:

1. Ensure you're using TypeScript 5.0 or later
2. Check that `moduleResolution` is set to `"bundler"` or `"node16"` in `tsconfig.json`
3. Restart your TypeScript server
